    1. **Create lib/notion-import.mjs** (per D-05, D-06):

    Imports: `{ existsSync, readFileSync, writeFileSync } from 'fs'`, `{ join, basename } from 'path'`, `{ createHash } from 'crypto'`, `{ mkdirp, ok, warn, info } from './shared.mjs'`, `{ readNotionConfig } from './notion-config.mjs'`, `{ cleanNotionFilename } from './docs.mjs'`.

    `contentHash(text)` — returns SHA-256 hex digest of text string. Uses `createHash('sha256').update(text, 'utf8').digest('hex')`.

    `stampFrontmatter(markdownBody, opts)` — prepends YAML frontmatter block:
    ```
    ---
    notion_page_id: {opts.page_id}
    notion_last_synced: {new Date().toISOString()}
    notion_content_hash: {contentHash(markdownBody)}
    ---

    {markdownBody}
    ```

    `parseFrontmatter(fileContent)` — extracts frontmatter fields + body:
    - If file starts with `---\n`, find closing `---\n`, parse key-value pairs
    - Return `{ notion_page_id, notion_last_synced, notion_content_hash, body }` (missing fields are undefined)
    - If no frontmatter, return `{ body: fileContent }`

    `importPage(vaultDocsDir, pageConfig, fetchedMarkdown)` — core import logic:
    - `vaultDocsDir` = `vault/projects/{slug}/docs/notion/` (caller computes, per D-04)
    - `pageConfig` = `{ page_id, page_url }` from notion_pages.json
    - `fetchedMarkdown` = raw markdown string from MCP fetch (caller provides)
    - Compute filename: `cleanNotionFilename(pageTitle + '.md')` where pageTitle is first `# heading` from fetchedMarkdown, or fallback to page_id
    - `mkdirp(vaultDocsDir)` to ensure notion/ subdir exists
    - Compute `newHash = contentHash(fetchedMarkdown)`
    - If target file does NOT exist: write with `stampFrontmatter(fetchedMarkdown, {page_id})`, `ok()` message, return `{status:'created'}`
    - If target file EXISTS:
      - `parseFrontmatter(readFileSync(target))` to get stored hash + body
      - If `newHash === stored.notion_content_hash`: no-op, `info('unchanged')`, return `{status:'unchanged'}`
      - Compute `localHash = contentHash(stored.body)`
      - If `localHash === stored.notion_content_hash`: no local edits, overwrite in place with new stamp, `ok('updated')`, return `{status:'updated'}`
      - If `localHash !== stored.notion_content_hash`: local drift detected per D-06, write `{filename}.notion-update.md` sibling with stamped new content, `warn('local edits detected')`, return `{status:'conflict'}`

    `importAllPages(projectDir, vaultPath, fetchFn)` — orchestrator:
    - Read config via `readNotionConfig(projectDir)`
    - If null, `fail('no .claude/notion_pages.json found')`, return
    - Compute `docsDir = join(vaultPath, 'docs', 'notion')` per D-04
    - For each page in config.pages: call `fetchFn(page.page_id)` to get markdown, then `importPage(docsDir, page, markdown)`
    - Return summary `{ created, updated, unchanged, conflict }` counts

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Notion URL input -> parseNotionUrl | User-supplied URL string, could be malformed or malicious |
| notion_pages.json -> readNotionConfig | User-editable JSON file on disk, could be corrupt |
| MCP markdown response -> importPage | External content from Notion MCP, could contain unexpected patterns |
| Filesystem write -> vault docs | Writing to user's vault; must not overwrite local edits |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | notion_pages.json | mitigate | validateNotionConfig rejects malformed JSON; page_id validated as 32 hex chars only |
| T-09-02 | Information Disclosure | parseNotionUrl | accept | URL parsing is local-only; no network calls; page_id is not secret (it's in the URL) |
| T-09-03 | Tampering | importPage file write | mitigate | 3-way hash check prevents overwriting local edits; writes to .notion-update.md sibling on drift (D-06) |
| T-09-04 | Denial of Service | importAllPages loop | accept | Page count bounded by user's own notion_pages.json; no external pagination |
| T-09-05 | Spoofing | MCP markdown content | accept | Content comes from user's own Notion workspace via authenticated MCP; no additional validation needed beyond hash tracking |
| T-09-06 | Tampering | frontmatter injection | mitigate | parseFrontmatter only extracts known keys (notion_page_id, notion_last_synced, notion_content_hash); unknown keys ignored |
</threat_model>
