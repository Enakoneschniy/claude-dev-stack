### 7. cds-quick (Phase 39) → /cds-quick skill dispatches agent

**Status: PARTIALLY BROKEN — DEMO-01 PIPELINE BREAK**

The `/cds-quick` skill was rewritten post-merge (commits `6d653fc`, `e05b9d2`) to use Claude Code's `Agent` tool directly instead of invoking `claude-dev-stack quick ... --json` via Bash.

**What this breaks:**
- The skill no longer calls `packages/cds-cli/src/quick.ts` → `dispatchAgent` → `CostTracker`
- The skill does NOT display cost (`cost_usd`) or `sessionId` to the user
- Auto-capture via the Stop hook is the only remaining session recording path — but the skill uses a sub-agent, not a full session, so whether the Stop hook fires for the outer session is session-architecture-dependent
- DEMO-01 success criterion 1 reads: "invokes `dispatchAgent`, the resulting session is auto-captured into SQLite, and the user sees a result summary **plus dollar cost**" — the cost display is broken

**What still works:**
- The `claude-dev-stack quick` CLI command is fully implemented and wired (`quick.ts` → `dispatchAgent` → `CostTracker` → cost output)
- The CLI path (invoked via `bin/cli.mjs`) does trigger `captureStandalone` when `CLAUDE_SESSION_ID` is absent

**Root cause:** Fix was applied to address a UX problem (Bash approval dialog + env var requirement), but was not scoped to DEMO-01 requirements. The CLI exists and works; the skill no longer exercises it.
