Exports to implement:

`validateScopes(obj)` -- returns `{ valid: boolean, reason?: string }`:
- `!obj || typeof obj !== 'object'` -> `{ valid: false, reason: 'not-an-object' }`
- `obj.version !== 1` -> `{ valid: false, reason: 'unknown-version' }`
- `!Array.isArray(obj.scopes) || obj.scopes.length === 0` -> `{ valid: false, reason: 'missing-scopes' }`
- `typeof obj.main_branch !== 'string'` -> `{ valid: false, reason: 'missing-main-branch' }`
- Otherwise `{ valid: true }`

`readScopes(projectDir)` -- reads `.claude/git-scopes.json`, returns parsed object or null:
- Path: `join(projectDir, '.claude', 'git-scopes.json')`
- If not exists, return null
- Parse JSON, validate via validateScopes, return object if valid, null if invalid

`writeScopes(projectDir, config)` -- writes config atomically:
- Path: `join(projectDir, '.claude', 'git-scopes.json')`
- Call `atomicWriteJson(path, config)`

`detectStack(projectDir)` -- returns `{ scopes: string[], confidence: 'high'|'medium'|'low', source: string }`:
- Check in order (highest signal first):
  1. `pnpm-workspace.yaml`: read file, regex `^\s*-\s*['"]?([^'"#\n]+?)['"]?\s*$` with `gm` flags on lines after `packages:`, expand each pattern via readdirSync direct children, return scope names as `basename(childDir)`
  2. `package.json` with `"workspaces"` array: parse JSON, expand each glob pattern the same way
  3. `lerna.json` with `"packages"` array: parse JSON, expand patterns
  4. `nx.json` exists: heuristic -- scan `apps/` and `packages/` dirs if they exist, confidence `'medium'`
  5. `turbo.json` exists: read `package.json` workspaces if present, expand patterns
  6. `Cargo.toml` with `[workspace]`: regex `members\s*=\s*\[([\s\S]*?)\]` to extract member paths, use `basename` of each
  7. Go multi-module: `readdirSync` looking for subdirs containing `go.mod`, each subdir is a scope
  8. `pyproject.toml` with `[tool.uv.workspace]`: regex `members\s*=\s*\[([\s\S]*?)\]`, expand patterns
  9. Fallback: `{ scopes: ['core'], confidence: 'low', source: 'fallback' }`
- For glob expansion (e.g. `apps/*`): split on `/`, take first segment as parent dir, `readdirSync(join(projectDir, parentDir), { withFileTypes: true }).filter(e => e.isDirectory()).map(e => e.name)`. Do NOT support `**` or nested globs.

`detectMainBranch(projectDir)` -- returns string or null:
- Step 1: `runCmd('git symbolic-ref refs/remotes/origin/HEAD --short', { cwd: projectDir })` -- returns e.g. `origin/main`, strip `origin/` prefix
- Step 2 (if step 1 null): `runCmd('git remote set-head origin --auto 2>/dev/null', { cwd: projectDir })` then retry step 1
- Step 3 (if still null): `runCmd('git branch --show-current', { cwd: projectDir })` -- proxy fallback
- Return null if all fail (caller handles prompt)

`installSkill(projectPath, config)` -- writes rendered SKILL.md:
- Read template from `join(PKG_ROOT, 'templates', 'skills', 'git-conventions', 'SKILL.md.tmpl')`
- Replace tokens:
  - `{{SCOPES_LIST}}` with `config.scopes.map(s => '- \`' + s + '\`').join('\n')`
  - `{{MAIN_BRANCH}}` with `config.main_branch || 'main'`
  - `{{TICKET_FORMAT}}` with `config.ticket_prefix ? '; ' + config.ticket_prefix + 'NNN' : ''`
  - `{{CO_AUTHORED_BY_SECTION}}` with `config.co_authored_by ? '- [ ] Co-Authored-By line included' : ''`
- Safety check: if `content.includes('{{')` throw `Error('SKILL.md template has unreplaced tokens')`
- Write to `join(projectPath, '.claude', 'skills', 'git-conventions', 'SKILL.md')` via `mkdirp(dirname(dest))` + `writeFileSync(dest, content, 'utf8')`

`printCommitlintInstructions(config)` -- prints install commands to console (never spawns npm):
- Print `npm install --save-dev @commitlint/cli@^19 @commitlint/config-conventional@^19 husky@^9`
- Print `npx husky init`
- Print `echo 'npx --no -- commitlint --edit "$1"' > .husky/commit-msg`
- Print generated `commitlint.config.mjs` content with `scope-enum` from `config.scopes` and `type-enum` from `config.types`

`createDefaultConfig(projectName, detected)` -- returns a default v1 config object:
```javascript
{
  version: 1,
  project: projectName,
  scopes: detected.scopes,
  types: ['feat', 'fix', 'refactor', 'test', 'docs', 'ci', 'chore'],
  ticket_prefix: '',
  ticket_regex: '',
  main_branch: 'main',
  branch_format: '{ticket}-{description}',
  commit_format: 'type(scope): subject',
  co_authored_by: false,
  commitlint_enforced: false,
  auto_detect: { enabled: true, sources: [detected.source] }
}
```

**Step 1: Create `lib/git-conventions.mjs`** (~150 lines)

Imports:
```javascript
import { existsSync, readFileSync } from 'fs';
import { join, basename } from 'path';
import { c, ok, fail, warn, info, prompt } from './shared.mjs';

  detectStack, readScopes, writeScopes, validateScopes,

  createDefaultConfig

```

`export async function main(args = [])`:
- Parse `args[0]` as subcommand
- `const isQuick = args.includes('--quick');`
- `const isFull = args.includes('--full');`
- Switch on subcommand:
  - `undefined`, `'help'`, `'--help'` -> `printHelp()`
  - `'list'` -> `cmdList()`
  - `'init'` -> `cmdInit({ quick: isQuick, full: isFull })`
  - `'refresh'` -> `cmdRefresh()`
  - `'add'` -> `cmdAdd(args.slice(1))`
  - `'remove'` -> `cmdRemove(args.slice(1))`
  - default -> `fail('Unknown scopes subcommand: ' + args[0]); printHelp();`

`function printHelp()`:
- Print usage: `claude-dev-stack scopes <command>`
- List subcommands: init, list, add, remove, refresh with descriptions

`async function cmdList()`:
- `const cwd = process.cwd();`
- `const config = readScopes(cwd);`
- If null: `fail('No .claude/git-scopes.json found in ' + cwd); return;`
- Print project name, scopes list, main branch, co_authored_by status

`async function cmdInit({ quick = false, full = false } = {})`:
- `const cwd = process.cwd();`
- `const projectName = basename(cwd);`
- Detect stack: `const detected = detectStack(cwd);`
- Detect main branch: `const mainBranch = detectMainBranch(cwd);`
- Build default config via `createDefaultConfig(projectName, detected)`
- If `mainBranch` found, set `config.main_branch = mainBranch`

Quick mode (`--quick` or default when not `--full`): 4 prompts via `prompt()`:
  1. Confirm project name (default: detected `projectName`)
  2. Confirm detected scopes: `"Detected scopes: [web, api, ui] (${detected.source}). Accept? (Y/n)"`
  3. Confirm main branch: `"Main branch: ${config.main_branch}. Correct? (Y/n)"`
  4. Commitlint: `"Install commitlint enforcement? (y/N)"` -- only if `existsSync(join(cwd, 'package.json'))`, default false

Full mode (`--full`): extends quick with additional prompts:
  5. Ticket prefix (default: empty)
  6. Branch format (default: `{ticket}-{description}`)
  7. Co-authored-by (default: false)

After prompts:
- `writeScopes(cwd, config);`
- `ok('Wrote .claude/git-scopes.json');`
- `installSkill(cwd, config);`
- `ok('Installed git-conventions skill');`
- If commitlint opted in: `printCommitlintInstructions(config);`

`async function cmdRefresh()`:
- Read existing config, re-detect stack, merge new scopes (add new, keep existing), write back
- Print diff: added scopes, unchanged scopes

`async function cmdAdd(args)`:
- Parse `args[0]` as scope name, validate non-empty
- Read config, push to scopes array, write back, reinstall skill

`async function cmdRemove(args)`:
- Parse `args[0]` as scope name
- Read config, filter out scope, write back, reinstall skill

## Trust Boundaries

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-05 | T (Tampering) | detectStack regex | accept | Regex only extracts directory names from controlled sentinel files; wizard confirms before writing; no code execution from extracted names |
| T-06-06 | I (Info Disclosure) | installSkill | mitigate | Safety check `content.includes('{{')` prevents unreplaced tokens leaking template internals to Claude session |
| T-06-07 | E (Elevation) | printCommitlintInstructions | mitigate | Never calls spawnSync('npm', ['install', ...]); only prints commands to console via info() |
| T-06-08 | T (Tampering) | git symbolic-ref | accept | Output is a branch name string; used only for default prompt value, user confirms; no shell injection (passed through runCmd which uses execSync with hardcoded command) |
</threat_model>
