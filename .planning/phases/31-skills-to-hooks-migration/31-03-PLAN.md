```markdown
## Skills vs Hooks

When migrating Claude Code automation, decide:

| Choice | Use when | Cost | Examples |
|--------|----------|------|----------|
| Hook   | Deterministic logic — keyword match, regex, file check, no LLM judgment | ~0 tokens (silent unless emits) | dev-router (regex), project-switcher (registry lookup), git-conventions (regex) |
| Skill  | Requires LLM reasoning — synthesis, decision-making, multi-step planning | Tokens at every prompt that activates the skill | session-manager /end (writes session log with synthesized summary), gsd-* (planning), notebooklm (querying) |

### Migration guideline

Ask: "Could a 50-line shell or Node script do this?" If yes → hook.
If the work needs Claude to summarize, decide between options, or generate prose → skill.

### Phase 31 migrations (claude-dev-stack v0.12)

- `dev-router` skill → `hooks/dev-router.mjs` (UserPromptSubmit regex, additionalContext)
- `session-manager` start-path → `hooks/session-start-context.sh` (SessionStart, vault context load)
- `project-switcher` skill → `hooks/project-switcher.mjs` (UserPromptSubmit regex against project-map.json)
- `git-conventions` skill → `hooks/git-conventions-check.mjs` (PreToolUse Bash(git commit*) regex)

Out-of-scope (remain skills): gsd-*, research-*, notebooklm, obsidian — all require LLM.
```
