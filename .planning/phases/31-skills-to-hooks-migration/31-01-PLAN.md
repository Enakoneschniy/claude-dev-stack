```javascript
async function main() {
  let payload;
  try {
    const chunks = [];
    await new Promise((resolve) => {
      const timer = setTimeout(resolve, 500);
      process.stdin.on('data', (chunk) => chunks.push(chunk));
      process.stdin.on('end', () => { clearTimeout(timer); resolve(); });
      process.stdin.on('error', () => { clearTimeout(timer); resolve(); });
    });
    const raw = Buffer.concat(chunks).toString('utf8').trim();
    if (!raw) return;
    payload = JSON.parse(raw);
  } catch {
    return;
  }
  // business logic
  process.stdout.write(text);  // becomes additionalContext for UserPromptSubmit
}
main().catch(() => {}).finally(() => process.exit(0));
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User prompt → dev-router/project-switcher hooks | Untrusted text crosses regex match; hook output prepended to Claude's context |
| Bash command string → git-conventions-check hook | Untrusted string parsed for commit message extraction |
| vault/project-map.json → project-switcher | File on disk; user-controlled but trusted source (own machine) |
| .planning/config.json → git-conventions-check | File on disk; user-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | Tampering (prompt injection) | hooks/dev-router.mjs | mitigate | Hook output is plain text describing routing — does NOT pass user prompt verbatim to Claude. Routing hint is a fixed template. Test case verifies output never echoes raw user text |
| T-31-02 | Tampering (prompt injection) | hooks/project-switcher.mjs | mitigate | Project name match uses word-boundary regex AND validated against project-map.json keys (allowlist). Output template includes only the matched project name + path from the map (no raw prompt echo) |
| T-31-03 | Tampering (shell injection in commit message) | hooks/git-conventions-check.mjs | mitigate | Hook NEVER executes the commit command — it only reads stdin tool_input.command as a string and validates the embedded message. No shell exec with user data. Suggestion output is plain text, no command construction |
| T-31-04 | Denial of Service (ReDoS) | hooks/dev-router.mjs regex | mitigate | Use anchored, simple alternations like `/\b(build|fix|implement|...)\b/i` — NO nested quantifiers, NO backreferences. Test case with 10KB pathological input must exit < 100ms. Add explicit length cap: ignore prompts > 4096 chars (silent skip) |
| T-31-05 | Denial of Service (ReDoS) | hooks/git-conventions-check.mjs regex | mitigate | Conventional commits regex uses anchored alternation, no nested quantifiers. Message extraction caps at 1024 chars before regex test |
| T-31-06 | Information Disclosure | hooks/project-switcher.mjs | accept | Hook reveals absolute project paths to Claude. Acceptable: same info already in Claude's context via cwd, vault config, etc. |
| T-31-07 | Elevation of Privilege | git-conventions-check strict mode | mitigate | Strict mode (exit 2) blocks tool execution per Claude Code hooks contract. Hook itself does NOT execute git or any command — it only signals. Mode read from .planning/config.json (project-local, requires repo write access) |
| T-31-08 | Spoofing | All hooks | accept | Hooks trust stdin payload from Claude Code runtime. No auth needed — Claude Code is the only legitimate caller. Acceptable per Anthropic hooks contract |
</threat_model>
