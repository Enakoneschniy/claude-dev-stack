```javascript
/**
 * Import all pages from a Notion database into the vault docs directory.
 * Uses cursor-based pagination (Notion 100-page limit per request).
 *
 * Per D-09: fetchFn signature: async (databaseId, cursor) => { pages, has_more, next_cursor }
 * Per D-04: file naming via toSlug(title) + '.md', fallback to page_id + '.md'
 * Per D-05: delegates per-page write to importPage() for 3-way hash protection
 *
 * @param {string} databaseId
 * @param {string} vaultDocsDir - absolute path to vault/projects/{slug}/docs/notion/
 * @param {Function} fetchFn - async (databaseId, cursor) => { pages, has_more, next_cursor }
 * @returns {{ created: number, updated: number, unchanged: number, conflict: number, total: number }}
 */
export async function importDatabase(databaseId, vaultDocsDir, fetchFn) {
  mkdirp(vaultDocsDir);
  const counts = { created: 0, updated: 0, unchanged: 0, conflict: 0, total: 0 };

  let cursor = null;
  let hasMore = true;

  while (hasMore) {
    const batch = await fetchFn(databaseId, cursor);
    for (const page of batch.pages) {
      const filename = page.title
        ? toSlug(page.title) + '.md'
        : page.page_id + '.md';
      const pageConfig = { page_id: page.page_id, page_url: '' };
      // importPage derives filename from markdown heading; override by writing to named path
      const { status } = await importPage(vaultDocsDir, { ...pageConfig, _filename: filename }, page.markdown);
      counts[status] = (counts[status] || 0) + 1;
      counts.total += 1;
    }
    hasMore = batch.has_more;
    cursor = batch.next_cursor ?? null;
  }

  return counts;
}
```

NOTE: importPage() currently derives filename from markdown heading via cleanNotionFilename. To support explicit filename from toSlug (D-04), you need to modify importPage() to accept optional _filename in pageConfig OR write to the target path directly in importDatabase(). Check the actual importPage() implementation: it reads headingMatch from fetchedMarkdown. Since importDatabase has the page title separately, a cleaner approach:

In importDatabase, do NOT call importPage(). Instead, call the lower-level steps directly:
1. Compute filename = page.title ? toSlug(page.title) + '.md' : page.page_id + '.md'
2. Call importPage but know it will derive its own filename from markdown heading. PROBLEM: if markdown lacks heading, it uses page_id.

Correct approach (per D-04 intent): keep importPage as-is (it reads heading from markdown), and in importDatabase pass markdown with the title as heading. OR: make importDatabase stamp the title into markdown before calling importPage.

Simplest approach matching existing code: if page.markdown starts with a heading, importPage uses it. If not, inject `# ${page.title}` as first line when title is non-empty. This ensures importPage produces the right filename. For empty-title pages, importPage uses page_id (matches test expectation).
