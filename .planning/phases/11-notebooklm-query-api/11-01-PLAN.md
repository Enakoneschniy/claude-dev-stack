**Retry wrapper (per D-11 -- 2x, 1s->2s exponential backoff):**
Implement a local async retry loop inside `askNotebook`. Only retry on `NotebooklmRateLimitError`. On non-rate-limit errors, throw immediately. Max 2 retries (3 total attempts). Delays: `1000 * Math.pow(2, attempt)` ms where attempt starts at 0 -> delays of 1000ms, 2000ms.

```javascript
let lastErr;
for (let attempt = 0; attempt <= 2; attempt++) {
  try {
    const parsed = runNotebooklm(args, { jsonMode: true, functionName: 'askNotebook' });
    // validate + map + return below
  } catch (err) {
    if (err instanceof NotebooklmRateLimitError && attempt < 2) {
      lastErr = err;
      await new Promise(r => setTimeout(r, 1000 * Math.pow(2, attempt)));
      continue;
    }
    throw err;
  }
}
throw lastErr;
```

**Step 2 -- For text types, download to temp file and read content (VERIFIED from Python source):**

CRITICAL: `notebooklm download` writes to a FILE on disk and returns JSON:
`{operation: 'download_single', artifact: {id, title, selection_reason}, output_path: '/path/to/file.md', status: 'downloaded'}`

It does NOT return content inline in the JSON. The content is in the FILE at `output_path`.

```javascript
let content = null;
if (!BINARY_ARTIFACT_TYPES.has(type)) {
  const tmpDir = mkdtempSync(join(tmpdir(), 'notebooklm-'));
  try {
    const dlArgs = ['download', type, '--json', '--latest', '-n', notebookId];
    const dlResult = run(dlArgs, {
      jsonMode: true,
      functionName: 'generateArtifact',
      spawnOpts: { cwd: tmpDir },
    });
    // dlResult shape: {operation, artifact:{id,title}, output_path, status}
    const filePath = dlResult?.output_path;
    if (filePath) {
      content = readFileSync(filePath, 'utf8');
    }
  } finally {
    rmSync(tmpDir, { recursive: true, force: true });
  }
}
return { artifactId, content, type };
```

Note: `runNotebooklm` uses `spawnSync` internally. It must pass `cwd` through to `spawnSync` options. Check if `runNotebooklm` already accepts `spawnOpts` or `cwd` in its options. If not, add `cwd` pass-through -- this is a one-line change in `runNotebooklm`:

const spawnOpts = { stdio: 'pipe', encoding: 'utf8', ...(options.spawnOpts || {}) };

|----------|-------------|
| JS wrapper -> notebooklm-py CLI | User question and notebook ID cross into external process via argv array |
| notebooklm-py stdout -> JS parsing | Untrusted JSON from subprocess parsed into return values |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

</threat_model>
