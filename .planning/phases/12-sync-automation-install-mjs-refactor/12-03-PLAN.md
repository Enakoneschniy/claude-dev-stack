Rewrite `bin/install.mjs` to be a thin orchestrator (~80-100 lines max, per D-04). The file must:

1. **Keep the shebang** `#!/usr/bin/env node` on line 1
2. **Import from lib/install/*.mjs** — all 13 modules via static imports (relative paths from bin/ are `../lib/install/xxx.mjs`)
3. **Import `{ prompt }` from `../lib/shared.mjs`** — needed for the "Ready to start?" confirmation prompt that lives in main()
4. **Import Node builtins** needed by the orchestrator itself: `{ resolve, dirname, join }` from `path`, `{ fileURLToPath }` from `url`, `{ homedir }` from `os`, `{ spawnSync }` from `child_process` (for vault git sync block at ~line 1418)
5. **Define PKG_ROOT** as `resolve(dirname(fileURLToPath(import.meta.url)), '..')`
6. **Define skillsDir/agentsDir** as `join(homedir(), '.claude', 'skills')` / `join(homedir(), '.claude', 'agents')`

The `main()` function preserves the EXACT flow from the original lines ~1319-1460:
- `printHeader()`
- "Ready to start?" confirm prompt (stays in orchestrator — it's the entry gate)
- `earlyTotal = '...'` (the "..." placeholder before step count is known)
- `checkPrerequisites(earlyTotal)` → `{ pythonCmd, pipCmd }`
- `collectProfile(earlyTotal)` → `profile`
- `collectProjects(earlyTotal)` → `projectsData`
- `projectsData._profileName = profile.name` (this assignment stays in orchestrator)
- `selectComponents(earlyTotal, !!pipCmd)` → `components`
- `selectAndInstallPlugins(PKG_ROOT, earlyTotal)` → `pluginResults`
- Step counting logic (calculating totalSteps from components booleans)
- Sequential calls with step numbers: installVault, installGSD, installObsidianSkills, installCustomSkills, installDeepResearch, installNotebookLM, installGitConventions, generateClaudeMD, installSessionHook
- `printSummary(...)` call
- Vault git sync block (the `spawnSync('git', ...)` section from ~line 1418) — this stays in the orchestrator since it's a one-off integration step, not a wizard section

**CRITICAL (Pitfall 6)**: The original line ~1418 does `const { spawnSync: spawn } = await import('child_process')` — convert this to use the static `spawnSync` import at the top. No dynamic import of builtins.

**Exports:**
```javascript
export { installNotebookLM } from '../lib/install/notebooklm.mjs';
export default main;
```

The re-export of `installNotebookLM` preserves backward compatibility for any code that imports from `bin/install.mjs` directly (per D-10). The `default` export is `main` (the orchestrator function), called by `bin/cli.mjs`.

**Target: under 100 lines total** (imports + constants + main + exports). The original main() was ~140 lines of sequential calls + step counting — with extracted modules this becomes ~60-70 lines of import-call-pass-result chains.
