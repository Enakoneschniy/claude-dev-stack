<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user slash-command `/budget-continue` → skill execution | User invokes skill; skill reads no untrusted input directly |
| AskUserQuestion → Claude → tool invocation | User's menu selection drives which tool is called |
| Bash (git remote get-url origin) → RemoteTrigger body | Git remote URL is passed as tool parameter |
| /schedule command → Desktop task daemon | Local scheduling surface; CC-managed |
| RemoteTrigger → Anthropic cloud API | OAuth-authenticated network call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-04 | Tampering | Git remote URL injection into RemoteTrigger body | mitigate | Skill uses `git remote get-url origin` via Bash — git CLI escapes shell metacharacters natively. Value is passed to RemoteTrigger as a structured parameter (not shell-interpolated). No additional sanitization needed. |
| T-21-05 | Information Disclosure | Skill leaks session state or secrets via prompt | mitigate | Skill prompts contain only hardcoded strings (`"/gsd-resume-work"`, `"/budget-continue"`). No interpolation of session_id, tokens, or user context into tool calls. |
| T-21-06 | Elevation of Privilege | RemoteTrigger invoked by API-key user (no cloud permission) | mitigate | Skill body explicitly notes: "RemoteTrigger uses claude.ai OAuth (not API key). If authenticated via API key only, surface error." Allows graceful failure over silent denial. |
| T-21-07 | Denial of Service | CronCreate storm (repeated invocations) | accept | Skill is invoked by user action only. Each invocation creates one one-shot timer. User must manually re-run to create more. Low risk. |
| T-21-08 | Repudiation | Scheduled task action attributed wrongly | accept | `/gsd-resume-work` and `/budget-continue` are both audit-logged by CC runtime (session transcripts). No additional attestation needed for ASVS L1. |
| T-21-09 | Spoofing | Malicious `.claude/loop.md` in downloaded repo runs untrusted code | mitigate | LIMIT-03 UAT verifies `diff` between template and installed file — confirms wizard writes the shipped template byte-for-byte, not arbitrary content. Installation is user-initiated only. |

All threats are ASVS L1 compliant. No high-severity threats block execution.
</threat_model>
