    1. Validate args:

       - `args.scope` validation:
         - If undefined: treat as 'current'.
         - If 'current' or 'all': valid.
         - Else: call `assertValidScopeBasename(args.scope)` from shared.ts (rejects `..`, `/`, `\\`, leading `~`, leading `/`, empty).

    2. Resolve vault path:
       - `const vault = deps?.vaultPath ?? process.env.CDS_TEST_VAULT ?? path.join(homedir(), 'vault');`
       - Verify `vault/projects` exists (statSync + isDirectory), else `throw new VaultNotFoundError(\`vault/projects not found at \${vault}\`)`.

    3. Resolve search paths:
       - `scope === 'current'`: project basename from `process.cwd()`. Search `{vault}/projects/{basename}/docs`.
       - `scope === 'all'`: search `{vault}/projects/*/docs` (glob expansion — pass multiple paths to rg, or iterate projects).
       - Else (specific basename): `{vault}/projects/{scope}/docs`.
       - For each resolved path, verify it exists (isDirectory). If none exist → return `{ hits: [], total: 0 }`.
       - Safety: `path.resolve(searchPath)` must start with `path.resolve(vault, 'projects')` as prefix. Otherwise `throw new InvalidFilterError('Path traversal detected')`.

    4. Detect ripgrep (cache on module-level `let hasRg: boolean | undefined`):
       ```ts
       function hasRipgrep(): boolean {
         if (hasRg !== undefined) return hasRg;
         const r = spawnSync('rg', ['--version'], { stdio: 'ignore' });
         return (hasRg = r.status === 0);
       }
       ```

    5. Build command:
       - ripgrep: `rg --json --no-heading --color never --type md --context 2 <query> <...paths>`
       - POSIX grep fallback: `grep -rnE -C 2 --include='*.md' <query> <...paths>`. Note: multi-path grep works fine; normalize output.

    6. Spawn (use `child_process.spawn`, not spawnSync — stream stdout):
       ```ts
       import { spawn } from 'node:child_process';
       import { createInterface } from 'node:readline';
       ```
       Collect stdout line-by-line. Stream-parse per §3.1-§3.3 of 37-RESEARCH.md.

    7. Normalize hits:
       - For ripgrep JSON: parse each line as JSON. Track current file via `begin`/`end` events. On `match`, emit hit with `context_before` collected from prior `context` events, `context_after` from subsequent `context` events until next file boundary or match.
       - For POSIX grep: parse lines by separator `--`. Lines with `:N:` are matches, `-N-` are context. Group by file.
       - In both cases, convert absolute path to vault-relative: `path.relative(vault, absPath)`.

    8. Apply limit at end: `hits.slice(0, clampedLimit)`.

    9. Error handling:
       - If rg fails (non-zero exit but no matches is code 1, which is OK; >1 is failure): swallow code 1, throw `InvalidFilterError` on codes ≥2 with stderr content.
       - If grep fallback also fails (e.g., bad regex): throw `InvalidFilterError` with stderr.

    10. Return `{ hits, total: hits.length }`.

    Do NOT log the spawned command in normal operation. Log a single warning on fallback:
    ```
    console.warn('ripgrep not found, using POSIX grep (slower on large vaults)');
    ```
    Log AT MOST ONCE per process (gate with `let loggedFallback = false`).
