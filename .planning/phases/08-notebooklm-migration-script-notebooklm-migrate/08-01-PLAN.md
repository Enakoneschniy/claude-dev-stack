Function signature per research:
```javascript
export async function migrateVault(opts = {}) {
  // opts.vaultRoot — defaults to findVault()
  // opts.sharedNotebookName — defaults to 'claude-dev-stack-vault'
  // opts.dryRun — defaults to true (D-07)
  // opts.delayMs — defaults to 1500 (D-08)
  // Returns: { dryRun, sources: [...], phaseAFailures, phaseBSkipped, orphans }
}
```

Implementation per decisions D-01 through D-08:

1. **Resolve shared notebook:** Call `listNotebooks()` once, find by `opts.sharedNotebookName` title match. Throw if not found.

2. **List sources:** Call `listSources(sharedNotebookId)` to get all sources from shared notebook.

3. **Parse and group:** For each source, call `parseSourceTitle(title)` to extract `{ slug, localTitle }`. The function uses `indexOf('__')` to find the first `__` separator, then `slice` to split: `slug = title.slice(0, idx)`, `localTitle = title.slice(idx + 2)`. Sources without `__` separator return `null` — these are orphans (D-04), mark `skipped_orphan` in log.

4. **Reverse mapping — resolve disk path from source title:** Import `_walkProjectFiles(vaultRoot)` from `lib/notebooklm-sync.mjs`. Call it once at startup to build a Map of `{slug}/{basename}` -> `filePath`. For each parsed source: look up `join(slug, localTitle)` in the map to get the disk path. If not found, mark as `failed` with reason `'file_not_found_on_disk'`. This is the critical link between NotebookLM source titles and vault files.

5. **Dry-run mode (D-07):** If `opts.dryRun === true`, print grouped output:
   ```
   cds__alpha (N sources):
     file1.md, file2.md, ...
   Orphans (no project prefix, will be skipped): K
   Total: X sources -> Y projects, K orphans
   Run with --execute to migrate
   ```
   Return results without any mutations. Use `info()`, `warn()`, `ok()` from shared.mjs.

6. **Load/create migration log (D-06):** Read existing `~/vault/.notebooklm-migration.json` via `existsSync` + `readFileSync`, or initialize empty `{ sources: [] }`. Build a Map keyed by `source_id` for O(1) resume lookup.

7. **Phase A — per-source upload+verify loop (D-01, D-02, D-05):**
   For each non-orphan source:
   a. Check resume: if existing log entry has `status === 'verified'` or `status === 'deleted'`, skip.
   b. Call `_ensureNotebook('cds__' + slug)` to get target notebook ID (cache per slug — call once per project).
   c. **Duplicate check (D-05):** Call `listSources(targetNotebookId)`, check if `localTitle` already exists. If yes, mark `verified` immediately, `atomicWriteJson` log, continue.
   d. **Upload:** Resolve disk path via the `_walkProjectFiles` lookup map built in step 4. Call `uploadSource(targetNotebookId, diskPath, { title: localTitle })`. If disk path was not found, this source was already marked `failed` in step 4.
   e. **Verify (D-02):** Call `listSources(targetNotebookId)`, find entry where `title === localTitle`. If found, mark `verified`. If not found, mark `failed`.
   f. Write migration log via `atomicWriteJson` after EACH state transition (D-06).
   g. `await sleep(opts.delayMs)` between operations (D-08).

8. **Phase B gate (D-03):** Count entries with `status === 'failed'`. If > 0, set `phaseBSkipped = true`, warn, return. `skipped_orphan` does NOT count as failure.

9. **Phase B — delete from shared (D-03):**
   For each entry with `status === 'verified'`:
   a. Call `deleteSourceByTitle(sharedNotebookId, entry.title)` wrapped in try/catch. Swallow `NotebooklmCliError` (source may already be gone per Pitfall 5 from research). Re-throw `NotebooklmRateLimitError`.
   b. Update entry status to `deleted`, `atomicWriteJson` log.
   c. `await sleep(opts.delayMs)`.

10. **Progress output:** Print `[N/total] uploading slug/localTitle...` per source during Phase A, and `[N/total] deleting title...` during Phase B.

Helper functions (internal, not exported):
- `parseSourceTitle(title)` — uses `indexOf('__')` + `slice` to return `{ slug, localTitle }` or `null` for orphans. Example: `'myproject__context.md'` -> `{ slug: 'myproject', localTitle: 'context.md' }`
- `sleep(ms)` — `new Promise(r => setTimeout(r, ms))`
- `migrationLogPath(vaultRoot)` — `join(vaultRoot, '.notebooklm-migration.json')`
- `readMigrationLog(vaultRoot)` — read or init empty log
- `writeMigrationLog(vaultRoot, log)` — `atomicWriteJson` wrapper
- `buildFilePathMap(vaultRoot)` — calls `_walkProjectFiles(vaultRoot)`, returns `Map<string, string>` keyed by `'{slug}/{basename}'` -> absolute disk path

     const vaultRoot = process.env.VAULT_PATH ?? findVault();
     if (!vaultRoot) {
       fail('Vault not found');
       info("Run: claude-dev-stack  (setup wizard to initialize your vault)");
       throw new Error('Vault not found');

     const { migrateVault } = await import('./notebooklm-migrate.mjs');
     try {
       const result = await migrateVault({ vaultRoot, dryRun });
       if (result.dryRun) {
         info('Run with --execute to migrate');
       } else {
         if (result.phaseBSkipped) {
           warn(`Migration incomplete — ${result.phaseAFailures} failure(s) in Phase A, shared notebook untouched`);

           ok(`Migration complete: ${result.sources.filter(s => s.status === 'deleted').length} sources migrated`);

       if (err instanceof NotebooklmNotInstalledError) {
         fail(err.message);
         info('Install with: pipx install notebooklm-py');
         throw err;
       }
       if (err instanceof NotebooklmRateLimitError) {
         warn('Rate limited — try again later');
         throw err;
       }
       throw err;
     }
   }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> migrateVault | User-supplied --execute flag controls mutation mode |
| migrateVault -> notebooklm-py subprocess | Untrusted subprocess output parsed for source IDs and titles |
| Migration log -> filesystem | Per-source state persisted atomically to vault root |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | Migration log `~/vault/.notebooklm-migration.json` | mitigate | Write via `atomicWriteJson` (temp+rename) to prevent partial writes on crash; read with JSON.parse in try/catch |
| T-08-02 | Information Disclosure | CLI output during migration | mitigate | Truncate source titles to 200 chars in progress output (reuse truncateReason pattern from notebooklm-cli.mjs T-05-01) |
| T-08-03 | Denial of Service | Rate limiting from notebooklm-py | mitigate | Fixed 1500ms delay between operations (D-08); re-throw NotebooklmRateLimitError to abort gracefully |
| T-08-04 | Elevation of Privilege | --execute flag accidentally true | mitigate | Explicit opt-in: `args.includes('--execute')` — default is dry-run (D-07). No mutation without explicit flag. |
| T-08-05 | Tampering | Shared notebook source deletion | mitigate | Phase B gate: zero Phase A failures required before any delete (D-03). Shared notebook never auto-deleted. |
| T-08-06 | Spoofing | Malicious source titles in shared notebook | accept | Titles are user-controlled via vault filenames; `parseSourceTitle` uses simple `indexOf('__')` split — no injection vector since titles are only used as string comparisons, never as shell args |
</threat_model>
