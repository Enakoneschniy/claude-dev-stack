## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-11 | Tampering (malicious session_id injection) | `${SESSION_ID:+--session-id "$SESSION_ID"}` in SKILL.md | mitigate | Bash parameter expansion with double-quotes prevents word-splitting and metachar interpretation. `adr-bridge-session.mjs` further validates session_id via regex (UUID shape) before using it in a path. If invalid, bridge falls back to mtime strategy. |
| T-26-12 | Tampering (prompt injection via transcript content) | Bridge Haiku call | mitigate | Covered in Plan 01 threat model (T-26-01). No new surface introduced by SKILL.md wiring. |
| T-26-13 | DoS (/end hangs on Haiku call) | SKILL.md bash block | mitigate | `adr-bridge-session.mjs` enforces 60s execFileSync timeout (Plan 01). Bash fallback `|| ADR_RESULT='...'` catches non-zero exit. /end never blocks indefinitely. |
| T-26-14 | Repudiation (ADR created under wrong project) | `$PROJECT_NAME` env var | mitigate | `validateProjectName` in bridge rejects path-traversal. `PROJECT_NAME` is derived from `basename $(git rev-parse --show-toplevel)` in SKILL.md — user-controlled but limited to basenames. |
