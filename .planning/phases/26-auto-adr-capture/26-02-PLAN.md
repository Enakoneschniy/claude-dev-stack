ADR file format variants to handle (from RESEARCH.md § "Existing ADR format"):

**Old format (0001-0012, no frontmatter):**
```markdown
# ADR-0005: Use pino for structured logging

**Дата**: 2026-03-15
**Статус**: accepted

## Контекст
...
```

**New format (created by Plan 01):** a YAML frontmatter block (delimited by triple-dashes) containing:

```markdown
... (yaml frontmatter start)
id: 0013
topic: logging-strategy
status: accepted
date: 2026-04-14
source:
  session_log: ...
  commit: a3f9c21
... (yaml frontmatter end)

# ADR 0013: Use pino for structured logging

## Context
...
```

CLI contract this plan creates:

```javascript
// lib/decisions-cli.mjs
export async function main(args): Promise<void>;

// Internal helpers (exported for testability):
export function parseAdrFile(filePath): {
  id: number,            // from frontmatter or filename NNNN prefix
  topic: string | null,  // from frontmatter if present, else null
  title: string,         // first H1/H2 line, stripped of "ADR-NNNN:" prefix
  status: string,        // from frontmatter `status:` or inline `**Статус**:` or 'unknown'
  date: string,          // from frontmatter `date:` or inline `**Дата**:` or ''
  topicFromFilename: string,  // always populated: filename minus NNNN- prefix and .md
  raw: string,           // full file content
};

export function listDecisions(decisionsDir): Array<parsed>;

export function findDecision(decisionsDir, query): parsed | null;
// query: '0005' | '5' | 'logging-strategy' | 'logging' (partial slug match)

export function searchDecisions(decisionsDir, term): Array<{ entry: parsed, score: number }>;
// Simple ranking: topic exact > title contains > content contains

export function resolveProject(cwd, explicitProject): {
  projectName: string,
  decisionsDir: string,
};
// If explicitProject given: vault/projects/{explicitProject}/decisions
// Else: derive from cwd (basename of git root or cwd) → vault/projects/{name}/decisions
// VAULT_PATH env var override same as other CLIs.
```

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → filesystem | `--project` flag controls which path under `vault/projects/` is read. |
| ADR file content → stdout | `show` and `search` echo raw file contents (could contain secrets if Plan 01 captured them). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-08 | Tampering (Path traversal via --project) | `resolveProject` | mitigate | `validateProjectName` rejects names matching `/[/\\]|\.\./` — same guard as `bridgeDecisions`. Tested by behavior Test 18. |
| T-26-09 | Information Disclosure (Reading ADRs of other projects) | `--project` flag | accept | Vault is user-local; the whole point of `--project` is cross-project browsing. User runs this on their own machine with their own vault — no elevation of privilege over running `cat vault/projects/*/decisions/*.md`. |
| T-26-10 | DoS (Enormous decisions dir slows `list`) | `listDecisions` | accept | Realistic cap: 1000 ADRs per project. Per-file read is O(n); parsing is trivial. No mitigation needed at this scale. |

</threat_model>
