<objective>
Add two new MCP tools to the CDS MCP server: `sessions.searchAll` (cross-project search via @cds/core searchAllProjects) and `memory.graph` (entity relationship graph via @cds/core getEntityGraph). Both tools are registered in the existing mcp-server.ts tool catalog and dispatched through the existing CallToolRequestSchema handler.
</objective>

<threat_model>
**T-45-01: Path traversal via memory.graph project param**
- Severity: HIGH
- Attack: Attacker passes `project: "../../etc"` to read arbitrary files
- Mitigation: Apply `assertValidScopeBasename()` from `shared.ts` to the `project` parameter before passing to `getEntityGraph()`
- Verification: Test that `memory.graph({ project: '../..' })` throws InvalidFilterError

**T-45-02: Unbounded result size from searchAll**
- Severity: LOW
- Attack: Query returns thousands of results consuming excessive memory
- Mitigation: Enforce `limit` parameter with max cap of 100 (same as sessions.search)
- Verification: Test that limit > 100 is clamped to 100
</threat_model>
