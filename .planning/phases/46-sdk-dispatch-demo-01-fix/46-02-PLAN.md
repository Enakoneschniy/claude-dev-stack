5. Update the cost display section. Change from:
```typescript

if (opts.json) {

  console.log(result.output);

# /cds-quick — One-shot quick task

Execute the user's task via the CDS CLI `quick` command. This routes through
`packages/cds-cli/src/quick.ts` → `dispatchAgent()` (SDK) for proper cost tracking.

Run this Bash command:

```bash
cd "$PROJECT_DIR" && npx claude-dev-stack quick '$ARGUMENTS'
```

If `npx claude-dev-stack` is not found (not installed globally or locally), try the dev path:

```bash
cd "$PROJECT_DIR" && node bin/cli.mjs quick '$ARGUMENTS'
```

Display the full CLI output to the user, including the cost line.

## When to use vs not

Use `/cds-quick` for:
- Short one-shot questions with cost tracking
- Tasks where you want SDK-level dispatch and cost_usd reporting
- Quick lookups, summaries, drafts

Do NOT use for:
- Multi-turn debugging (use Claude Code normally)
- Code edits requiring file context (use Edit/Write tools directly)
- Tasks that need tools (quick dispatch has no tool access by default)
```

- Added fallback to `node bin/cli.mjs` for dev mode
