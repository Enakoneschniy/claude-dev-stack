Create `packages/cds-core/src/vault/multi-search.ts` with the following structure (per D-06, D-07, D-08):

1. Module comment: `// Phase 43 — Cross-project search via SQLite ATTACH batching (MEM-02).`

2. Imports:
   ```typescript
   import { existsSync, readdirSync } from 'node:fs';
   import { homedir } from 'node:os';
   import { basename, join } from 'node:path';
   import Database from 'better-sqlite3';
   ```

3. Type exports:
   ```typescript
   export interface CrossSearchHit {
     project: string;          // basename of project directory
     observationId: number;
     sessionId: string;
     type: string;
     content: string;
     entities: number[];
     createdAt: string;
     sessionSummary: string | null;
     rank: number;             // BM25 rank from FTS5 (lower = better match)
   }
   ```
   NOTE: CrossSearchHit is a flat interface (Claude's discretion — flat is simpler than extending SearchHit which has nested `observation` object). The `project` field identifies which project vault the hit came from.

4. Private helper `discoverProjectDbs(projectsDir: string): Array<{ path: string; project: string }>`:
   - `readdirSync(projectsDir)` — catch ENOENT and return `[]`
   - For each entry, check `existsSync(join(projectsDir, entry, 'sessions.db'))`
   - Return `{ path: join(projectsDir, entry, 'sessions.db'), project: entry }` for each match

5. Private row interface:
   ```typescript
   interface SearchRow {
     id: number;
     session_id: string;
     type: string;
     content: string;
     entities: string;      // JSON string from DB
     created_at: string;
     session_summary: string | null;
     rank: number;
   }
   ```

6. Private helper `runBatchSearch(dbPaths: Array<{ path: string; project: string }>, query: string, limitPerProject: number): CrossSearchHit[]`:
   - Open primary DB: `new Database(dbPaths[0].path, { readonly: true })`
   - Set `db.pragma('query_only = ON')` for safety
   - ATTACH remaining DBs in batch: `db.prepare('ATTACH ? AS p' + j).run(dbPaths[j+1].path)` for j=0..batch.length-2
   - For EACH db in the batch (primary + attached), run a SEPARATE FTS5 query and collect results. Do NOT try to UNION FTS5 across schema aliases (per RESEARCH.md Pitfall 4 / Assumption A2). Instead:
     - For primary (index 0): query the main schema directly using the existing FTS5 pattern from sessions.ts
     - For attached (index j, schema alias `p{j}`): run a separate query against `p{j}.observations_fts`
   - The per-schema query pattern:
     ```sql
     SELECT o.id, o.session_id, o.type, o.content, o.entities, o.created_at,
            s.summary AS session_summary, bm25({schema}.observations_fts) AS rank
     FROM {schema}.observations_fts
     JOIN {schema}.observations o ON o.id = {schema}.observations_fts.rowid
     LEFT JOIN {schema}.sessions s ON s.id = o.session_id
     WHERE {schema}.observations_fts MATCH ?
     ORDER BY rank LIMIT ?
     ```
     Where `{schema}` is `main` for primary, `p0`, `p1`, ... for attached DBs.
   - Map each SearchRow to CrossSearchHit with the correct `project` name. Parse `entities` JSON string: `JSON.parse(row.entities) as number[]`
   - Wrap in try/finally to guarantee `db.close()`

7. Main export `searchAllProjects(query: string, options?: { vaultPath?: string; limit?: number }): CrossSearchHit[]`:
   - Resolve vault root: `options?.vaultPath ?? process.env['VAULT_PATH'] ?? join(homedir(), 'vault')`
   - Discover DBs via `discoverProjectDbs(join(vaultRoot, 'projects'))`
   - If no DBs found, return `[]`
   - Batch in groups of 9 (ATTACH limit is 10 per connection; primary uses 1 slot, leaving 9): `for (let i = 0; i < dbPaths.length; i += 9)`
   - Each batch: `runBatchSearch(dbPaths.slice(i, i + 9), query, limitPerProject)`
   - Merge all batch results, sort by rank ascending (lower BM25 = better match)
   - Slice to overall limit: `options?.limit ?? 100`
   - Return sorted `CrossSearchHit[]`

| T-43-07 | Denial of Service | multi-search.ts many projects | accept | Bounded by local filesystem. Batch size of 9 limits open connections. Each batch connection closed in finally block. |
