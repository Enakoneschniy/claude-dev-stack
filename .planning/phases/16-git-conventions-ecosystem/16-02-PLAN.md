  '    branches:', '      - ' + mainBranch (concatenation, not template literal),

  '      - uses: actions/setup-node@v4', '        with:', '          node-version: 20',
  '      - name: Install commitlint', '        run: |',
  '          npm install --save-dev @commitlint/cli@^19 @commitlint/config-conventional@^19',
  '      - name: Validate commit messages', '        run: |',
  then the npx commitlint line with the SHA references included as literal text in the string,
  and a final empty string. Return array.join('\n').

  For the npx line with SHA references: write it as a plain string literal where the
  dollar-sign-brace sequences are just characters in the string. In a regular string (not a
  template literal), dollar-sign has no special meaning, so write:
    '          npx --no -- commitlint --from ${{ github.event.pull_request.base.sha }} --to ${{ github.event.pull_request.head.sha }} --verbose'
  This is a plain string -- no escaping needed because it is not inside backticks.

  parseClauda implementation:
  Guard: return empty result for falsy or non-string input.

  P2: Use headingRegex = /^#{1,4}\s+(scopes|packages|modules|services|components)\s*$/im
  exec on content; if match, slice content after match end, split by newline, collect bullet
  items via /^[-*]\s+`?(\w[\w-]*)/ until next heading line.

  P3 (if P2 found nothing): match /\bscopes?\s*:\s*([^\n]+)/i; split the capture on /[,\s]+/;
  filter to /^\w[\w-]*$/ identifiers.

  P4 (if P3 found nothing): count backtick-quoted identifiers matching /`(\w[\w-]{1,20})`/g;
  collect those appearing 2 or more times.

  Post-process: deduplicate via Set, lowercase, replace spaces with hyphens, slice to 30.

  types: filter ['feat','fix','refactor','test','docs','ci','chore'] to those whose word boundary
  regex matches in content.

  mainBranch: try /\bmain\s+branch\s*:?\s*([\w/-]+)/i then /\bbranch\s*[:\s]+\b(main|master|develop|\w+)\b/i.

  ticketPrefix: try /\bticket\s+prefix\s*:?\s*([A-Z]{2,6}-)/i then
  /\b([A-Z]{2,6})-\d+\b[\s\S]{0,200}\b\1-\d+\b/ for repeated ticket pattern.
  Ensure returned prefix ends with '-'.
