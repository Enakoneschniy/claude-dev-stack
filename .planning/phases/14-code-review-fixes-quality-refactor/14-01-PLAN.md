Fix all 4 warnings per D-01, D-02, D-03 (no behavior changes — defensive fixes only):

**WR-01 (unused tmpdir in generateArtifact):** In `lib/notebooklm.mjs`, function `generateArtifact` around line 717 creates `const tmpDir = mkdtempSync(join(tmpdir(), 'notebooklm-'))` but the subsequent `notebooklm download` command writes to its own `output_path` (determined by notebooklm-py), NOT into our tmpDir. The tmpDir is created, never populated, then cleaned up — pure waste. Remove the `mkdtempSync` call, the `rmSync(tmpDir, ...)` in the finally block, and simplify the try/finally to a plain block. If `tmpdir` import from `node:os` is no longer used anywhere in the file after this change, remove it from the import statement. Check if `mkdtempSync` is still needed (it IS used in `uploadSource` around line 420) — keep it in the import if so.

**WR-02 (missing null check on dlResult):** In the same `generateArtifact` function, after the download call `const dlResult = run(dlArgs, ...)`, the code does `const filePath = dlResult?.output_path` which handles null dlResult, but then `readFileSync(filePath, 'utf8')` will throw if filePath is undefined/null. Add a guard: only call `readFileSync` when `filePath` is a non-empty string AND the file exists (use `existsSync`). Add `existsSync` to the `node:fs` import if not already there. This makes the function resilient to notebooklm-py returning a download result without an output_path field.

**WR-03 (shell quoting in display):** In `lib/notebooklm-cli.mjs`, function `runGenerate` around line 352, the info message constructs a shell command suggestion: `info(\`Binary download to ${filepath} -- run: notebooklm download ${typeArg} -n ${notebookId} -o "${filepath}"\`)`. The filepath is already double-quoted in the template literal, but the outer quoting is inconsistent — if filepath contains a double-quote character itself it would break. Wrap filepath in single quotes instead of double quotes in the display string: `-o '${filepath}'`. This is a display-only fix (the string is printed for the user to copy-paste, not executed).

**WR-04 (silent flag discard):** In `lib/notebooklm-cli.mjs`, functions `runSync(_subArgs)` (line 71) and `runStatus(_subArgs)` (line 425) accept sub-arguments but never inspect them. If a user passes `notebooklm sync --dry-run`, the flag is silently ignored. Add a simple unknown-flag warning at the top of each function: parse `_subArgs` (rename to `subArgs`), check for any args starting with `--` or `-`, and if found call `warn(\`Unknown flag ignored: ${flag}\`)` for each. Do NOT throw — just warn. This preserves forward-compatibility (future flags won't crash old versions).
