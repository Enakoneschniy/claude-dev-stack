Create `lib/project-naming.mjs` following established ESM patterns (see lib/shared.mjs for style):

```javascript
/**
 * lib/project-naming.mjs — centralized project slug utilities.
 *
 * Consolidates the toLowerCase().replace(...) chain duplicated across
 * add-project.mjs, docs.mjs, templates.mjs, import.mjs, install/projects.mjs.
 */
```

Export two named functions:

1. `toSlug(name)` — Per D-05 spec exactly:
   `name.toLowerCase().replace(/\s+/g, '-').replace(/[^a-z0-9-]/g, '').replace(/-{2,}/g, '-').replace(/^-|-$/g, '')`
   Note: the existing codebase pattern does NOT have the consecutive-hyphen collapse or leading/trailing strip. The D-05 spec adds them. This is safe because the existing pattern's output is a subset of the new one's output (existing never produced consecutive hyphens from spaces, but could from special chars like "a!@b" -> "a-b" vs "a!!b" -> "ab" — both are fine).

2. `fromSlug(slug, vaultPath)` — Per D-06: scan `join(vaultPath, 'projects')` directory, find entry matching slug. Return the directory name if found, null otherwise. Use `readdirSync` + `existsSync` from `fs`. Import only what's needed.

- Test all behavior cases listed above for toSlug
- Test fromSlug with a temp directory structure (use `mkdtempSync` + `mkdirSync` to create fake vault/projects/my-project)
- Test fromSlug returns null for non-existent slug
- Clean up temp dirs in test teardown
