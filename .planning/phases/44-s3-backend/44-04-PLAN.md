<action>
Create `packages/cds-s3-backend/src/merge.test.ts`:

Tests create real SQLite databases in a temp directory, populate them with different sessions, and verify merge correctness.

Key test cases:

1. **Two-device merge: different sessions**
   - Device A: session "aaa-aaa" with 2 observations, 1 entity "typescript"
   - Device B: session "bbb-bbb" with 3 observations, 2 entities "typescript" (same), "react" (new)
   - After merge B into A: A has both sessions, 5 total observations, 2 entities
   - "typescript" entity exists once (INSERT OR IGNORE deduplication)

2. **Same sessions on both devices**
   - Both have identical session UUIDs
   - After merge: no new rows added

3. **First sync (empty local)**
   - Local DB has no sessions, remote has 3 sessions
   - After merge: local has all 3 sessions with all observations

4. **Schema version mismatch**
   - Remote has higher user_version than local
   - mergeRemoteIntoLocal throws S3SchemaVersionError

5. **Entity ID remapping**
   - Device A: entity "react" has id=1. Device B: entity "react" has id=5
   - Observation on B references entity id=5 in its entities JSON
   - After merge: observation's entities JSON contains local entity id for "react" (id=1)

6. **Relations merge with ID remapping**
   - Device B has relations referencing its local entity IDs
   - After merge: relations use remapped local entity IDs

Helper: create test DBs by reading migration SQL files from cds-core package (use relative path `../../cds-core/src/vault/internal/migrations/`).

Use `mkdtempSync(join(tmpdir(), 'cds-merge-test-'))` for test isolation.
Clean up temp directories in afterEach with `rmSync(dir, { recursive: true })`.
</action>
