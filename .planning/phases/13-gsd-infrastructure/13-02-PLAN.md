    ```bash

    ```

    **Invoke bridge:**

    ```javascript
    // Run as inline ESM — transition.md operates within Claude Code's Node.js context
    import { findVault } from './lib/projects.mjs';
    import { bridgeDecisions } from './lib/adr-bridge.mjs';

    const vaultPath = findVault();
    const contextPath = `.planning/phases/${completed_phase_slug}/${completed_phase}-CONTEXT.md`;

    try {
      const result = await bridgeDecisions({
        phaseNumber: completed_phase,
        phaseName: completed_phase_name,
        phaseSlug: completed_phase_slug,
        contextPath,
        vaultPath,
        projectName: PROJECT_NAME,
      });

      if (result.action === 'created') {
        info(`ADR bridge: wrote ${result.adrPath} (${result.decisionCount} decisions)`);
      } else if (result.action === 'noop') {
        info('ADR bridge: ADR already exists for this phase, skipping');
      }
      // 'skipped' is silent — no vault or no decisions
    } catch (err) {
      warn(`ADR bridge failed (non-blocking): ${err.message}`);
    }
    ```

    ```markdown
    **Parallel phase detection (before presenting next phase):**

    Check if multiple independent phases can run concurrently.

    ```bash
    ROADMAP_DATA=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" roadmap analyze)
    ```

    Parse the JSON result. For each phase with status "pending" (not started, not complete):
    1. Extract its `depends_on` list (phase numbers it depends on)
    2. A phase with "Nothing" or empty depends_on has no dependencies

    **Independence test:** Two pending phases A and B are independent if:
    - A's number is NOT in B's depends_on list
    - B's number is NOT in A's depends_on list
    - Neither A nor B depends (transitively) on a phase that is not yet complete

    If 2+ independent pending phases are found:

    **Calculate cost estimate (per D-09):**
    - Per-phase token estimate: plan_count x 15k tokens (rough average)
    - Total: sum of all parallel phase estimates
    - Wall-clock comparison: sequential = sum, parallel = max

    **Present consent prompt (per D-10):**

    Use `AskUserQuestion` with this format:

    ```
    ## Parallel Execution Available

    Found {N} independent phases that can run concurrently:
    - Phase {A}: {name} ({X} plans, ~{Ak}k tokens)
    - Phase {B}: {name} ({Y} plans, ~{Bk}k tokens)

    Estimated total: ~{T}k tokens
    Wall-clock: ~{parallel_min} min parallel vs ~{sequential_min} min sequential

    Options:
    1. Run in parallel (via TeamCreate with worktree isolation)
    2. Run sequentially (default — phases in numbered order)
    ```

    **If user selects "Run in parallel" (per D-13):**

    Spawn parallel executors using TeamCreate:

    ```
    Use TeamCreate to spawn {N} team members:
    - Member 1: Execute `/gsd-execute-phase {A}` with isolation: "worktree"

    ```

    Each member gets its own worktree to prevent file conflicts between phases on different branches.

    - Collect results from all members
