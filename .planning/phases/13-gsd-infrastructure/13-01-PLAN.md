    - `bridgeDecisions({ phaseNumber, phaseName, phaseSlug, contextPath, vaultPath, projectName })` — main function.

    bridgeDecisions logic:
    1. Validate inputs: throw on missing phaseNumber/contextPath. Path traversal guard on projectName (per T-01 from session-context.mjs pattern, line 126-128).
    2. If vaultPath is null/undefined or doesn't exist: return `{ action: 'skipped', reason: 'no vault' }`.

    7. Generate slug from phaseSlug or phaseName: lowercase, replace spaces/underscores with hyphens, strip non-alphanumeric except hyphens. Example: "gsd-infrastructure" stays as-is.
    8. Build ADR content (per D-02, D-04, D-06):
       ```
       # ADR-{NNNN}: Phase {N} — {Phase Name}

       ---
       source: gsd-bridge
       phase: {N}
       decision_ids: [D-01, D-02, ...]
       ---

       **Date**: {YYYY-MM-DD}
       **Status**: accepted
       **Generated by**: gsd-bridge from .planning/phases/{slug}/CONTEXT.md

       ## Context

       GSD phase discussion for "{Phase Name}" produced {count} locked decisions.
       Source: .planning/phases/{phaseSlug}/{phaseNumber}-CONTEXT.md

       ## Decisions

       {For each decision: ### D-{NN}: {text}}

       ## Consequences

       Decisions captured automatically from GSD discuss-phase. Individual decisions
       may be superseded by later phases or manual ADRs.
       ```
    9. Write file atomically (tmp + rename pattern from session-context.mjs).
    10. Return `{ action: 'created', adrPath: filePath, decisionCount: decisions.length }`.
