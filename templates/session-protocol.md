# Session Protocol

Правила работы Claude Code с базой знаний. Этот файл подключается через CLAUDE.md каждого проекта.

## При старте сессии

1. Прочитай `context.md` текущего проекта из vault
2. Прочитай последние 3 файла из `sessions/` чтобы понять что делалось недавно
3. Если задача затрагивает другой проект — прочитай его `context.md` тоже
4. НЕ спрашивай "с чего начнём?" — у тебя уже есть контекст, предложи продолжение

## Во время работы

1. Если принимается архитектурное решение (выбор библиотеки, изменение структуры, новый подход) — сразу запиши ADR в `decisions/`
2. Если обнаружен баг или workaround — отметь в `context.md` в секции Known Issues
3. Если паттерн может быть полезен другим проектам — добавь в `shared/patterns.md`

## При завершении сессии

1. Создай файл `sessions/YYYY-MM-DD-краткое-описание.md` с summary:
   - Что было сделано (конкретные файлы, фичи)
   - Что осталось (TODO на следующую сессию)
   - Проблемы и решения
   - Изменения в архитектуре (если были)
2. Обнови `context.md` если изменился статус, стек или текущие задачи
3. Если менялся стек или зависимости — обнови секцию Stack в context.md

## Формат ADR (Architecture Decision Record)

Файл: `decisions/NNNN-краткое-название.md`

```markdown
# ADR-NNNN: Название решения

**Дата**: YYYY-MM-DD
**Статус**: accepted | superseded | deprecated

## Контекст
Почему возник вопрос

## Решение
Что выбрали

## Альтернативы
Что рассматривали и почему отвергли

## Последствия
Что это меняет в проекте
```
